# X2DFD QA Accuracy Evaluation

## Overview
This project simulates question answering for face forgery detection datasets. The pipeline loads questions alongside real and fake image metadata, generates placeholder answers via a pluggable inference module, reports dataset-level accuracy, ranks per-question scores, and stores every intermediate artefact in timestamped run folders.

## Project Structure
- `data/` – input JSON files (`question.json`, `real.json`, `fake.json`).
- `qa_utils/` – shared helpers:
  - `loader.py` – question/image metadata readers.
  - `evaluation.py` – accuracy computation, question ranking, and response aggregation.
  - `inference.py` – stubbed model inference (random `real`/`fake`) and narrative generation.
  - `persistence.py` – result-directory management and logging utilities.
- `qa_accuracy.py` – main entry point orchestrating the simulation, inference, and persistence workflow.
- `results/` – autogenerated outputs:
  - `run_log.csv` – append-only manifest of runs (timestamp, run_id, seed, top_k, accuracies).
  - `latest_run.json` – pointer to the most recent run and its artefacts.
  - `runs/run_<timestamp>/` – per-run folder containing:
    - `settings.json` – configuration snapshot (inputs, seed, top_k).
    - `dataset_metrics.json` – real/fake evaluation metrics with per-question stats and full response narratives.
    - `per_question_balanced.json` – balanced accuracy for every question in the dataset.
    - `question_ranking_all.json` – full ranking of questions ordered by balanced accuracy.
    - `question_ranking_top{k}.json` – ordered list of the retained top-k questions.
    - `concatenated_questions_top{k}.txt` – concatenated question string for the ranking.
    - `responses_real.json` / `responses_fake.json` – per-image, per-question responses with narratives for each dataset split.
    - `summary.json` – quick summary linking all artefacts and headline metrics.

## Usage
```bash
python3 qa_accuracy.py [--top-k N]
```
The script prints dataset accuracies, balanced accuracy, and saves all intermediate artefacts under a dedicated `results/runs/run_<timestamp>/` directory. Use `--top-k` to control how many top-ranked questions are retained (default: 3).
